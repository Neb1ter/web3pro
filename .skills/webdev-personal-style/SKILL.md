# 个人建站习惯知识库

> **每次开始新建站或编辑网站任务时，必须先询问用户**：「是否采用已学习的建站习惯知识库？」确认后按本规范执行。

---

## 一、设计风格偏好

### 整体风格
- 深色背景为主（深蓝 `#0A192F` / 深黑 `#050D1A`），搭配高饱和度主题色
- 每个页面/板块有**独立主题色**，颜色与内容主题强关联（不用通用蓝/灰）
- 卡片使用深色半透明背景，圆角 `rounded-2xl`，悬停时边框发光
- 避免纯白背景；文字颜色需与背景形成足够对比度

### 主题色分配原则
- 每个独立功能板块分配一个专属主题色，全站所有 UI 元素（按钮/边框/图标/进度环/浮动菜单）严格跟随该板块主题色
- 品牌色（如第三方产品官方色）不属于主题色，需单独保留，不受主题色替换影响

### 视觉质感
- 重要按钮和交互元素使用**泛光效果**（`drop-shadow` + `glow`）
- 进度/状态类元素使用渐变色或发光环
- 背景可使用微妙的网格纹理或渐变噪点，避免纯色平铺

## 二、交互与导航习惯

### 长页面必备组件（内容超过 3 屏的页面必须包含）

1. **进度环回顶按钮**（右下角 `fixed bottom-6 right-4`）
   - SVG 进度环实时显示阅读百分比，泛光质感
   - Tooltip 显示「已读 XX% · 点击回顶」
   - 滚动超过 300px 后显示，颜色跟随页面主题色
   - 必须放在主组件 return 最外层（不能放在子组件内）

2. **可拖拽浮动章节菜单**（左下角 `fixed bottom-6 left-4`，与回顶按钮同一水平线）
   - 支持鼠标拖拽和触控拖拽，避免遮挡内容
   - 展开时高斯模糊背景（`backdrop-blur-md`）
   - 触发按钮显示当前章节图标 + 简短提示文字（如「点此切换章节」）
   - 颜色跟随页面主题色

3. **上一篇/下一篇导航**（多章节内容页底部）
   - 减少用户回到侧边栏重新选择的操作
   - 点击后自动滚回顶部

### 内容组织方式
- 多内容板块优先使用 **Tab 切换**，而非长页面堆叠
- 章节顺序遵循**自然阅读流**：从「是什么」→「为什么」→「怎么做」→「行动/CTA」
- 复杂功能抽离为**次级页面**，主页面只做引导入口（如「不知道怎么做？点这里」）
- 跨页面内容关联时，在当前页面底部加**跳转提示卡片**，告知用户相关内容在哪里

## 三、移动端优化（重点，用户极为重视）

### 顶部导航栏（长页面专用）
- 结构：`[← 返回主页] | [章节标题（移动端）/ 页面标题（桌面端）]`
- 移动端滚动时，当前章节标题动态滑入导航栏「返回主页」按钮右侧
- 章节切换时用 `slideInFromBottom` 动画替换（`key={activeChapter}` 触发）
- 返回主页按钮：移动端只显示箭头图标（文字 `hidden sm:inline`）
- 桌面端保持不变，只在移动端（`sm:hidden`）显示章节标题

### 字体排版
- 正文：`text-sm`（移动端）/ `text-base`（桌面端）
- 标题：`text-xl sm:text-2xl`，避免移动端字号过大
- 行高：`leading-relaxed`（1.625），减少阅读疲惫感
- 段落间距：`mb-4 sm:mb-6`；卡片内边距：`p-4 sm:p-6`

### 浮动元素位置
- 浮动菜单：`bottom-6 left-4`；回顶按钮：`bottom-6 right-4`（同一水平线）
- 两者不遮挡主要内容区域

## 四、内容设计习惯

### 降低用户恐惧感（面向新手内容时）
- 先用**类比**或**对比表格**建立认知（如「XX vs 传统方式」）
- 提供**官方数据背书**（权威机构数据、审计报告）增加信任感
- 设置**模拟体验**（计算器、模拟器）让用户无风险理解概念
- 避免直接堆砌专业术语，先科普再深入

### 信息架构
- 首页/入口页：引导型，CTA 明确，不堆砌信息
- 内容页：章节化，每章节有独立 ID（用于锚点和浮动菜单定位）
- 工具页：交互优先，计算器/对比表等实用工具放在显眼位置
- 次级页面必须有明确的返回入口（左上角返回按钮）

---

## 五、技术实现偏好

- **动画**：偏好 CSS `@keyframes`，不引入额外动画库
- **组件复用**：通用组件通过 `color` / `theme` prop 适配不同主题，抽离到 `components/`
- **状态提升**：子组件需要的状态若影响多个组件，提升到父组件
- **滚动感知**：用 `getBoundingClientRect` 或 `IntersectionObserver` 检测当前可见章节
- **高斯模糊**：浮动面板、模态框使用 `backdrop-blur-md`
- **重写策略**：复杂页面重写时可不参考原有代码，按需求重新设计结构
